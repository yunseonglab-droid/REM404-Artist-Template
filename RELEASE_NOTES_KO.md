# REM404 Artist Template v1.3.0

## v1.3.0 — GitHub 커밋 완료 전 편집 잠금

- 앱에서 전시 정보를 저장하면 고유한 동기화 버전을 생성합니다.
- GitHub Actions가 해당 버전의 설정을 커밋할 때까지 전시 정보 입력란을 잠급니다.
- 앱을 닫거나 다른 관리자가 접속해도 Firebase에 저장된 잠금 상태가 유지됩니다.
- 앱은 15초마다 GitHub에 실제 커밋된 JSON 스냅샷을 확인합니다.
- 커밋 버전이 일치하면 잠금이 자동으로 해제되고 다시 수정할 수 있습니다.
- 자동 커밋 확인 주기를 10분에서 5분으로 단축했습니다.
- `지금 확인` 버튼으로 커밋 상태를 즉시 다시 확인할 수 있습니다.

## v1.2.0 — 앱과 전시 정보 자동 동기화

## 주요 변경 사항

- 전시 생성 기능을 관리자 앱에서 제거하고 GitHub Pages 전시 등록 방식으로 통일했습니다.
- REM404 앱에서 수정한 전시명, 영문명, 설명, 전시 기간과 운영 상태가 관객용 페이지에 즉시 반영됩니다.
- GitHub Actions가 5분마다 공개 전시 설정을 확인하고 변경된 설정을 자동 커밋합니다.
- Firebase 연결이 일시적으로 실패하면 GitHub에 저장된 마지막 설정 스냅샷을 사용합니다.
- 전시 기간이 끝나면 관객 페이지에서도 종료 상태를 자동으로 표시합니다.
- GitHub 개인 액세스 토큰을 앱이나 정적 페이지에 저장하지 않는 안전한 동기화 구조를 적용했습니다.

## 사용 방법

1. 이 저장소로 새 전시 저장소를 만듭니다.
2. 인식 이미지와 공개 이미지를 교체한 후 GitHub Pages를 활성화합니다.
3. REM404 관리자 앱에서 `전시 등록하기`를 눌러 GitHub Pages QR 또는 주소를 등록합니다.
4. 전체 관리자가 등록을 승인합니다.
5. 전시 관리 화면에서 정보를 수정하면 관객 페이지에 즉시 반영되고, GitHub에는 자동 스냅샷 커밋이 남습니다.

## 확인 사항

- 저장소의 Actions 권한에서 워크플로의 콘텐츠 쓰기가 허용되어 있어야 자동 커밋이 가능합니다.
- 이미 만들어진 이전 전시 저장소에는 v1.3.0 동기화 파일을 한 번 적용해야 합니다.
- 앱과 정적 페이지에는 GitHub PAT 또는 서버 비밀키를 입력하지 않습니다.
